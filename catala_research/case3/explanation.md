# Формализация модели объёмов СОИ (вода) — пояснительная записка

Формализация **вычислимой** части модели начисления объёмов коммунальной услуги на
содержание общего имущества (СОИ) по холодному водоснабжению, горячему
водоснабжению и холодному водоснабжению для нужд ГВС.

Источник: `Модель_начисления_объемов_услуги_на_СОИ.md`. Правовая база:
ПП РФ №354 от 06.05.2011 (формулы 11–13 Приложения №1) и ПП РФ №306 от 23.05.2006
(Приложение №1 п.5 — норматив 0,0903 м³/чел).

Модель считает **объём** (в м³) ресурса на СОИ, приходящийся на i-е помещение за
расчётный период. Денег в модели нет: снижение платы за некачественную услугу — это
отдельный денежный контур рантайма, к объёму не относится (см. ниже).

## Файлы

- `soi_volume.catala_en` — модель (модуль `Soi_volume`).
- `soi_volume_tests.catala_en` — Catala-тесты (`#[test]`-скоупы, быстрый контур).
- `test_soi_volume.py` — pytest поверх транспилированного Python (**приёмочный
  критерий** — проверяется тот код, что поедет в рантайм).
- `build_pyrun.sh` — сборка standalone Python-пакета под `_build/pyrun/soi`.
- `Makefile`, `clerk.toml` — тулчейн и корень clerk-проекта.

## Как запускать

```
make typecheck   # проверка типов модели
make test        # Catala-тесты (assertion-based; провал => exit 123)
make python      # транспиляция модели в Python -> _build/python/Soi_volume.py
make python-pkg  # сборка импортируемого пакета _build/pyrun/soi (модель+stdlib+runtime)
make pytest      # pytest поверх собранного Python-пакета
make clean       # очистка _build
```

Всё воспроизводится с `make clean` (первый `make test`/`make pytest` пересобирает
нативный stdlib через prereq `setup`).

## Основные функции (скоуп → пункт документа)

| Скоуп | Что считает | Формула |
|---|---|---|
| `NoMeterSOI` | Объём СОИ на i-е помещение **без ОДПУ** | `V_i = N_ои × S_ои × (S_i / S_об)` |
| `MeteredColdSOI` | То же **с ОДПУ, холодная вода** | `V_i = (V_д − ΣV_нж − ΣV_жн − ΣV_жп − V_гв − V_то) × (S_i/S_об)` |
| `MeteredHotSOI` | То же **с ОДПУ, горячая вода** | `V_i = (V_д − ΣV_нж − ΣV_жн − ΣV_жп − V_эг) × (S_i/S_об)` |
| `NormativeCap` | Нормативный потолок объёма по дому | `min(норматив_субъекта_на_м² × S_ои, 0.0903 × persons)` |

Вспомогательные функции: `area_share(S_i, S_об)` — доля с защитой от деления на
ноль; `prorate(value, сутки_периода, сутки_месяца)` — пропорция месячного норматива
по суткам.

В метерных скоупах порядок такой: (1) сырой объём по дому = показание ОДПУ минус все
отдельно учтённые потребления, с клэмпом `< 0 ⇒ 0`; (2) `house_soi = min(сырой,
нормативный_потолок)`; (3) `V_i = house_soi × доля_площади`.

## Параметры (провенанс) — зашиты в модели

Все нормативы — в одной структуре `Norms` и одной константе `const_norms`:

| Поле | Значение | Единицы | Источник |
|---|---|---|---|
| `per_person_norm` | `0.0903` | м³/чел/мес | ПП РФ №306, Прил.1 п.5 (федеральный, фиксированный) |
| `subject_norm_per_m2` | `0.03` *(пример)* | м³/м²/мес | норматив субъекта РФ (для потолка) |
| `n_oi` | `0.03` *(пример)* | м³/м²/мес | N_ои — норматив на СОИ (формула без ОДПУ) |

`subject_norm_per_m2` и `n_oi` — **региональные** величины; заданы примерными
значениями. Каждый скоуп получает `context terms`, по умолчанию `const_norms`,
поэтому другой регион/дом обслуживается тем же кодом переопределением `terms` (это
проверяет `test_terms_override_changes_norm`). Пиннятся версии норматива, а не
значения — для объяснимости и ретро-пересчётов.

## Входы, которые заполняет рантайм (не Catala)

Catala отвечает за формулы; данные приходят на вход. Все объёмы — в м³.

**Геометрия дома** (паспорт МКД, реестр помещений):
- `area_i` (S_i) — площадь i-го помещения;
- `area_common` (S_ои) — площадь помещений общего имущества (МОП);
- `area_total` (S_об) — суммарная площадь всех жилых и нежилых помещений;
- `persons` — число жителей (для подушевого потолка 0,0903).

**Календарь периода** (для пропорции при частичном периоде):
- `days_in_period`, `days_in_month` — сутки в расчётном периоде и в месяце.

**Показания приборов учёта за период** (метерные скоупы):
- `v_house` (V_д) — показание ОДПУ;
- `sum_nonresidential` (ΣV_нж) — нежилые помещения (п.43 Правил);
- `sum_residential_no_meter` (ΣV_жн) — жилые без ИПУ;
- `sum_residential_metered` (ΣV_жп) — жилые с ИПУ (п.59 Правил);
- `v_hot_selfproduced` (V_гв), `v_heating` (V_то) — только холодная вода;
- `v_energy` (V_эг) — только горячая вода.

В pytest все эти входы **замоканы** конкретными значениями под граничные случаи.

## Зафиксированные интерпретации двусмысленностей

Каждая отражена в прозе `.catala_en` и покрыта граничным тестом:

1. **Пропорция по суткам** применяется к нормативно-выведенным месячным величинам
   (объём по формуле без ОДПУ и оба предела потолка), а показания приборов уже
   относятся к фактическому периоду и повторно **не** пропорционируются.
   *(тесты `TestNoMeterProration`, `TestCapProration`)*
2. **Нормативный потолок** применяется к объёму по дому в метерных формулах
   (`min(сырой, потолок)`), реализуя общее правило «объём СОИ не может превышать
   нормативных величин». В формуле без ОДПУ результат уже нормативный по построению,
   поэтому потолок к нему не применяется. *(тесты `TestColdCapBinds`,
   `TestCapSubjectBinds` / `TestCapPersonBinds` — оба исхода `min`)*
3. **Клэмп неотрицательности**: если сумма вычитаемого превышает показание ОДПУ,
   объём СОИ по дому равен 0 (не может быть отрицательным). *(тесты
   `TestColdNegativeClamp`, `TestHotNegativeClamp`)*
4. **Деление на ноль**: при нулевой суммарной площади доля = 0; при некорректном
   календаре (`сутки_в_месяце ≤ 0`) пропорция не применяется. *(тесты
   `TestNoMeterZeroArea`, `TestNoMeterZeroDaysInMonth`)*

## Инварианты

- Объём СОИ по дому неотрицателен (клэмп `< 0 ⇒ 0`).
- Объём СОИ по дому не превышает нормативного потолка (в метерных формулах).
- Доля площади определена при любой суммарной площади (0 при `S_об ≤ 0`).

## Прозаические (невычислимые) пункты — вне модели

- *Снижение платы при некачественной/непредоставленной услуге* применяется к
  **плате**, а не к объёму — денежный контур рантайма; объём здесь номинальный.
- *«минусовой прибор учёта»* — это словесное описание метода вычитания, уже
  реализованного метерной формулой; отдельного правила не требует.
- *Состав `S_ои`* (лестничные площадки, коридоры, холлы, помещения охраны и т.п.) —
  дефиниция площади из паспорта дома, не формула; `area_common` приходит готовым.

## Что проверяют тесты

13 pytest-тестов (и зеркально — Catala `#[test]`-скоупы) покрывают:
- обе формулы без/с ОДПУ (холодная и горячая вода), базовые расчёты;
- оба исхода нормативного потолка (`min`): связывает подушевой предел и предел
  субъекта РФ;
- пропорцию по суткам (частичный период);
- клэмп неотрицательности при «минусовом» результате;
- защиту от деления на ноль (нулевая площадь, нулевой календарь);
- переопределение `terms` (переиспользуемость: другой регион = другой норматив).

Все результаты — точные рациональные величины (Catala считает `decimal` точной
дробью), поэтому равенства в тестах строгие.
